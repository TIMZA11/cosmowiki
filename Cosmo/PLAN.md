# 🌌 CosmoWiki — План проекта

## Концепция
Веб-сайт-каталог космических объектов с автоматическим получением данных из Википедии.
Тёмная космическая тема, вдохновлённая дизайном DesignWave (скриншот заказчика).

---

## 1. Анализ вариантов архитектуры

### Вариант A: Чистый фронтенд (HTML + CSS + JS)
- **Плюсы:** Не нужен сервер, можно открыть как файл или захостить на GitHub Pages
- **Минусы:** CORS-ограничения API Википедии (но MediaWiki API поддерживает CORS через `origin=*`), вся логика на клиенте
- **Вердикт:** ✅ ПОДХОДИТ — Wikipedia API поддерживает кросс-доменные запросы

### Вариант B: Node.js бэкенд (Express) + фронтенд
- **Плюсы:** Серверное кэширование, обход CORS, предобработка данных
- **Минусы:** Нужен Node.js, усложнение деплоя
- **Вердикт:** Избыточен для текущей задачи

### Вариант C: Python бэкенд (Flask) + фронтенд
- **Плюсы:** Отличные библиотеки для Wikipedia (wikipedia-api)
- **Минусы:** Нужен Python-сервер
- **Вердикт:** Избыточен

### ✅ ВЫБОР: Вариант A — чистый фронтенд
**Почему:** Wikipedia REST API (`ru.wikipedia.org/api/rest_v1/`) и MediaWiki API поддерживают CORS.
Сайт можно просто открыть в браузере без сервера. Минимальная сложность, максимальная простота деплоя.

## 4. Источники данных (приоритет)

### 1. NASA (основной источник)
```
GET https://images-api.nasa.gov/search?q={query}&media_type=image
```
- Фотографии космических объектов (бесплатно, без ключа, CORS)
- Маппинг 70+ объектов на англоязычные поисковые термины NASA
- Полноразмерные изображения через `/asset/{nasa_id}`

### 2. Wikipedia (дополнительный источник)
```
GET https://ru.wikipedia.org/api/rest_v1/page/summary/{title}
GET https://ru.wikipedia.org/w/api.php?action=parse&page={title}&prop=wikitext&format=json&origin=*
```
- Описания объектов (на русском языке)
- Параметры из инфобоксов
- Fallback-изображения (если NASA не нашёл)

### 3. Naked Science (дополнительный источник)
```
https://naked-science.ru/article/top/gid-po-vselennoi-kto-est
```
- Описания категорий на русском
- Иллюстрации для страниц категорий

## 5. Расчёт времени полёта

Формула на основе текущих технологий:
- **Ближайшие объекты (Солнечная система):** скорость ≈ 60 000 км/ч (New Horizons — самый быстрый зонд)
- **Дальние объекты (звёзды, галактики):** скорость ≈ 60 000 км/ч + примечание о нереальности полёта

Данные о расстояниях берём из Wikipedia или храним предзаданные значения.

## 9. Ключевые решения

1. **SPA vs MPA:** Используем MPA (3 HTML-файла) с передачей параметров через URL query strings (`?category=planets`, `?object=mars`). Проще, не нужен роутер.

2. **Кэширование:** Данные из Wikipedia кэшируем в `localStorage` на 24 часа, чтобы не нагружать API и ускорить повторные загрузки.

3. **Fallback данные:** Для каждого объекта храним базовые данные (расстояние, тип), чтобы страница работала даже при недоступности API.

4. **Время полёта:** Считаем по формуле `расстояние / скорость`, где скорость = 58 536 км/ч (скорость «Вояджер-1» — самого далёкого космического аппарата).
